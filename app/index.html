2002 - First job, first challenge... Flash

I got my first job in 2002, 2nd year of study. It was at a local TV station that hired me based on my Flash multimedia cd. I had to build a Flash based application that would load custom built ads, the news, the weather and everything in between.

Adobe Photoshop

What were my tasks?

Cropping, slicing, color correction, building new templates for the ads and working with clients that wanted a custom built advert… I’ve done them all.

These were the days that Photoshop became the de facto design tool for me, and stays this way even today. It was the tool that I also used for my web projects back in 2007.

Macromedia Flash

What did I have to build?

An application that loaded ads, dynamically- they were loaded based on certain criteria, but most of the time the clients that choose the top advert package were first.

And since we’re talking advert packages, there were 3 – complex, medium and small. All of them had to be built from the ground up – design, animation and scripting, as per the client’s request and business profile.

XML

How did I use it?

The news desk had 2 jobs – get fresh news off the field and insert them in an xml file with a specific format that I would later parse and insert the data in the News section of the project.

2007 - Flash is dead, long live HTML, CSS and JavaScript

Back in 2007, it was clear that Flash had a limited time span. It was time to learn the technologies that would allow Google to index web sites properly. So there I was, from a career that was entirely Flash based to a career in frontend development.

HTML

Switching from Flash to HTML?

The transition was a bit awkward – no more drag and draw rectangles on the timeline and then add key frames and Actionscript to do their magic.

Instead, I had to start thinking in terms of blocks or divs which would hold content and that content had other divs inside it. But once I got past the whole floating concept, HTML became easy.

CSS

Write once, apply everywhere!

There’s not much to say about CSS, except I fell in love with it immediately. The fact that I could divide my styles into separate files, modules as I like to call them, was excellent.

During the following years, I kept thinking about the fact that I could not do animations with it. Along came CSS3 – key frames and animations!

Javascript&Jquery

Thinking about animation in the DOM?

If you’ve worked with Flash, right clicking and inserting key frames, then linking those together and creating your animation was excellent.

However, doing that exact thing in Javascript meant that I had to work in terms of DOM elements –target them first and then manipulate them.

Jquery came to the rescue, reducing the amount of code I had to write In Javascript.

2012 - School's out...

In 2009 I decided to study computer science. There, I fell in love with Java and object oriented programming. At the end of 2011, I started working on my dissertation, which was built using Java, MySQL & JDBC.

JAVA SE 6

How about some OOP?

Getting to work with Java was mind blowing. It was like my mind was designed to work with objects. I had colleagues who just could not get it and asked me to do their homework for them.

When the opportunity to create a Java application for a clinic presented itself, I jumped right in. Unfortunately, after graduation, the firm that i landed a job at did not have any Java projects whatsoever, so my Java skills remained somewhere between novice and intermediate...

MySQL

Left Join, Right Join and the Like

It was when I started building my Java app that I first starting working with MySQL.

The experience I got was great, especially when you have to build something for the real world. However, writing those queries, that spanned multiple lines in JDBC, felt a little strange…

Hibernate

You wrote all those queries in JDBC?

That was a question that I received at a job interview. When I answered “Yes”, my interviewer said: “Brave man!”.

I did not get the job, but that’s how I learned about Hibernate and Object Relational Mapping.

I did use Hibernate in a couple of personal projects, but not at work, since no java projects existed. I’m hoping to change that in 2015.

2014 - Javascript everywhere...

…but most importantly on the server. Everyone I knew was talking about Node.js ,Express and MongoDB. Pair these javascript technologies with Google’s AngularJS and you had javascript on the frontend and the backend.

Node.JS

Javascript on the server?

During my stay at [e-spres-oh], I’ve met Andrei. Great guy and a great javascript developer. He kept talking about how Node is going to change everything.

I did lend an ear and after work started to play around with Node, tried Express, created some routes and I found myself working on a pretty complex app, that was using javascript on both ends – frontend and backend.

MongoDB

No more JOINS. How about some populations?

To be honest, I never enjoyed MySQL or the SQL syntax. So you can imagine the joy when I could write javascript code in Mongoose, the de facto ORM for MongoDB.

The trickiest part for me was understanding how to populate sub-documents and how to use the ObjectId property of models in order to create Many-To-Many relationships. With that out of the way, I’ve enlisted in MongoDB University certification program, that started on January 6th, 2015.

AngularJS

Dependency injection and project structure?

This was a very strange journey for me… and to most of my colleagues. The main problem was not the dependency injection concept, which can be found in the Java based Spring framework, but how to structure your project…

AngularJS Novice to Ninja, from Sitepoint, is an excellent book and I encourage you to read it if you haven’t done so already.

2015 - A fresh start

It will be a year of new challenges and since everything now is mobile first, I decided to give Swift a shot. More programming languages, maybe Haskell, will follow on the "wish list" this year.

Swift

How about those tuples?

My interest in learning Swift came with my first iPhone ever. I want to build a movie app, but since I’ve never built one before, it is time to learn the programming language that will allow me to do just that – build that app.

While reading an article on the net, I’ve found out about tuples in Swift and where they originate from – Haskell – so this will be my 2nd programming language to learn in 2015.

MY TEAM

When I started Globe Studios back in 2007, I was completely alone. It was a split moment decision that I should start my own company.

As more and more projects came my way, I was unable to honor them anymore – so I’ve started asking my friends and searching job portals. When I was done, the team of 2008-2010 consisted of Arnold, Ciprian, Simina and me.

2010 brought new faces to the table, while old ones had to go. Robert and Arjana joined the team, while Ciprian and Simina left.

Until its closure, Globe Studio’s team of 2010-2012 was formed by me, Arnold, Robert and Arjana.

MY FRIENDS

I’m proud to say that even after all this time, we’re still in touch. Arnold is one of my best friends and I’m happy to say that we can exchange, eventually, ideas and discussions about the backend. In the early days, my frontend work was voodoo magic for him, and his backend work was something similar for me.

As for Robert and Arjana, I just visited them in Scotland last year and had a great time doing so.

Ciprian has moved to Germany, but we did manage to bump into each other during the summer holidays. Simina is an art-director now, working for Fortune 500 companies.

OUR GOAL

The goal of each team was a simple one – embark on an ordinary project and make it something else, something that was unique. We’ve never used any templates or purchased themes, everything was started from scratch.

I’ve also believed, and still believe, that learning from each other and improving every day was a crucial goal that would allow us to work faster and better.
